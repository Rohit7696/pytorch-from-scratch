01

What Is Autograd?

Autograd (Automatic Gradient) is PyTorch's engine that records every operation on tensors into a computational graph, then uses the chain rule to compute derivatives automatically — exactly what gradient descent needs.

1

requires_grad=True

Tells PyTorch: "Track all operations on this tensor so I can compute gradients later."

2

Forward Pass

Perform normal math. PyTorch silently builds a directed acyclic graph (DAG) of every operation.

3

.backward()

Walks the graph backward, applying the chain rule. Stores results in each tensor's .grad.

x, w, b
requires_grad
→
z = wx + b
forward
→
ŷ = σ(z)
sigmoid
→
L(ŷ, y)
loss
←
∂L/∂w, ∂L/∂b
.backward()

Gradient Accumulation (The Gotcha)

Gradients accumulate by default — they don't overwrite. Each .backward() call adds to .grad. This is why you must zero them between training steps.

Quick Reference

Concept Code Purpose
Track tensor x = torch.tensor(1.0, requires_grad=True) Enable autograd for this tensor
Forward y = x**2 + 3*x Builds computation graph automatically
Backward y.backward() Compute ∂y/∂x via chain rule
Read gradient x.grad Access the computed gradient value
Zero gradients x.grad.zero_() Reset accumulated gradients
Disable tracking x.requires_grad_(False) Permanently disables tracking
Detach z = x.detach() Clone without grad history
No-grad context with torch.no_grad(): Disable tracking in a block (inference/updates)
→
Key takeaway: Autograd records the forward pass into a graph, then walks it backward using the chain rule to compute how much each input contributed to the output. That's exactly what gradient descent needs to train neural networks.
