Data Preprocessing

Drop unused columns, encode labels (M=1, B=0), split into train/test sets, and standardize features using StandardScaler (zero mean, unit variance).

Drop Columns

Remove id and Unnamed: 32 — they carry no predictive information.

Label Encoding

Convert diagnosis from M/B to 1/0 using LabelEncoder so the model can compute with it.

Train/Test Split

80% training, 20% testing with train_test_split. Stratified to maintain class balance.

Standard Scaling

Features have very different ranges (area ~1000, smoothness ~0.1). Scaling prevents large features from dominating.

Key Concepts Recap

Why StandardScaler?▸
Features like area_mean range in hundreds/thousands while smoothness_mean is below 0.2. Without scaling, large-valued features dominate gradient updates. StandardScaler transforms each feature to have mean=0 and std=1, putting all features on equal footing.
Why Sigmoid Activation?▸
Sigmoid squashes any real number to the range (0, 1), making the output interpretable as a probability. For binary classification, this is exactly what we need — the probability of the positive class (malignant). The formula is σ(z) = 1 / (1 + e⁻ᶻ).
Binary Cross-Entropy Loss▸
The loss function measures how far predictions are from true labels. Binary cross-entropy is: L = -[y·log(ŷ) + (1-y)·log(1-ŷ)]. When y=1, it penalizes low ŷ heavily; when y=0, it penalizes high ŷ. The torch.clamp with epsilon prevents log(0) = -infinity.
Why torch.no_grad() for Updates?▸
Weight updates (w -= lr * w.grad) are not operations we want to differentiate. Without torch.no_grad(), PyTorch would try to track these updates in the computational graph, wasting memory and potentially causing errors. The update step is a "plain assignment", not a learnable operation.
Limitations of This Model▸
This is a single-layer network (logistic regression). It can only learn linear decision boundaries. For more complex patterns, you'd add hidden layers with non-linear activations. The accuracy of ~63% shown in the reference content suggests the model would benefit from more epochs, tuning, or added complexity.
