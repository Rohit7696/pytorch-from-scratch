01

Why the nn Module Exists

In the previous notebook, we built a neural network from scratch — manually creating weights, writing the sigmoid, computing BCE loss, and updating parameters with raw autograd. The nn module replaces all of that with clean, reusable building blocks.

Without nn.Module (Manual)

Create weights with torch.randn(..., requires_grad=True)
Write sigmoid: torch.sigmoid(z)
Write loss: -(y*log(ŷ) + (1-y)*log(1-ŷ)).mean()
Update: w -= lr * w.grad inside no_grad()
Zero: w.grad.zero_(), b.grad.zero_()

~30 lines of boilerplate per model

With nn.Module (Clean)

Layers: self.linear = nn.Linear(30, 1)
Activation: self.sigmoid = nn.Sigmoid()
Loss: nn.BCELoss()
Optimizer: torch.optim.SGD(model.parameters(), lr)
Update: optimizer.step() (one line!)

Clean, modular, production-ready

02

Building Your First nn.Module Model

Every PyTorch model inherits from nn.Module. You define layers in __init__ and data flow in forward. That's it — two methods.

first_model.py
import torch
import torch.nn as nn

class Model(nn.Module):
    def __init__(self, num_features):
        super().__init__()
        self.linear = nn.Linear(num_features, 1)
        self.sigmoid = nn.Sigmoid()

    def forward(self, features):
        out = self.linear(features)
        out = self.sigmoid(out)
        return out

super().__init__()

Initializes the parent nn.Module class. Without this, nothing works — autograd tracking, parameter registration, .to(device), all depend on this.

nn.Linear(in, out)

Creates a fully connected layer with weight matrix (out × in) and bias vector (out). Automatically sets requires_grad=True on both. This replaces manual torch.randn + requires_grad.

nn.Sigmoid()

The sigmoid activation — same as torch.sigmoid() but as a layer object. Squashes values to (0, 1). Wrapping it as nn.Sigmoid() makes it part of the model's architecture.

forward(self, x)

Defines the data flow. You never call this directly — call model(x) instead. PyTorch's __call__ method handles hooks, autograd, and then delegates to forward.

→
Key insight: nn.Linear internally does x @ W.T + b — the exact same matmul we wrote manually. But it automatically creates, tracks, and registers the parameters.

hidden_layer_model.py
class Model(nn.Module):
    def __init__(self, num_features):
        super().__init__()
        self.linear1 = nn.Linear(num_features, 3)   # hidden: 3 neurons
        self.relu = nn.ReLU()                        # non-linearity
        self.linear2 = nn.Linear(3, 1)                # output: 1 neuron
        self.sigmoid = nn.Sigmoid()

    def forward(self, features):
        out = self.linear1(features)
        out = self.relu(out)
        out = self.linear2(out)
        out = self.sigmoid(out)
        return out
test_hidden.py
Why ReLU between layers?▸
Without a non-linear activation between linear layers, stacking them is mathematically equivalent to a single linear layer. W₂(W₁x + b₁) + b₂ simplifies to Wx + b. ReLU breaks this linearity, letting the network learn complex, non-linear decision boundaries.
Why 3 hidden neurons?▸
It's a starting point. More neurons = more capacity to learn, but also more risk of overfitting and more computation. For this demo with 5 features, 3 hidden neurons is reasonable. In practice, you experiment with different sizes (16, 32, 64, 128) and see what works.
Parameter count: 5→3→1▸
Linear1: (5×3) + 3 = 18 params. Linear2: (3×1) + 1 = 4 params. Total = 22 trainable parameters. Use torchinfo.summary() to verify. The ReLU and Sigmoid add zero parameters — they're just mathematical functions.

The old manual BCE with clamp is commented out. We'll use nn.BCELoss() instead — cleaner and numerically stable.

The Big Three — What Changed

Manual (Old Way)

loss = -(y*log(ŷ) + (1-y)*log(1-ŷ)).mean()

loss.backward()
with torch.no_grad():
  model.weights -= lr * model.weights.grad
  model.bias -= lr * model.bias.grad
model.weights.grad.zero_()
model.bias.grad.zero_()

nn.Module (New Way)

loss_fn = nn.BCELoss()
optimizer = torch.optim.SGD(model.parameters(), lr)

loss = loss_fn(ŷ, y)
optimizer.zero_grad()
loss.backward()
optimizer.step()

nn.BCELoss() — What it does▸
Binary Cross-Entropy Loss. Computes -[y·log(ŷ) + (1-y)·log(1-ŷ)] internally — the exact same formula we wrote manually. But it's optimized, numerically stable, and integrates cleanly with the optimizer. We no longer need the torch.clamp workaround.
torch.optim.SGD — The optimizer▸
SGD (Stochastic Gradient Descent) replaces the manual weight update. model.parameters() returns all tensors with requires_grad=True — the optimizer tracks and updates ALL of them. No more manually accessing model.weights.grad and model.bias.grad.
optimizer.zero_grad() → loss.backward() → optimizer.step()▸
The sacred three-line pattern. zero_grad() clears all gradients (replaces manual .grad.zero_() on every parameter). backward() computes gradients via autograd. step() updates all parameters (replaces the no_grad block). This is the standard training pattern used everywhere in PyTorch.
y_train_tensor.reshape(-1, 1)▸
BCELoss expects predictions and targets to have the same shape. The model outputs shape (455, 1) but y_train is (455,). reshape(-1, 1) or view(-1, 1) converts y_train to (455, 1) to match. Without this, you'd get a shape mismatch error.

The Complete Code — Copy & Run

Everything in one cell. Copy this into a Jupyter notebook or Colab and run.

complete_pipeline.py (copy this)
import numpy as np
import pandas as pd
import torch
import torch.nn as nn
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler, LabelEncoder

# ---- Load & Clean ----
df = pd.read_csv('https://raw.githubusercontent.com/gscdit/Breast-Cancer-Detection/refs/heads/master/data.csv')
df.drop(columns=['id', 'Unnamed: 32'], inplace=True)

# ---- Split ----
X_train, X_test, y_train, y_test = train_test_split(
    df.iloc[:, 1:], df.iloc[:, 0], test_size=0.2)

# ---- Scale & Encode ----
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test  = scaler.transform(X_test)

encoder = LabelEncoder()
y_train = encoder.fit_transform(y_train)
y_test  = encoder.transform(y_test)

# ---- Tensors ----
X_train_tensor = torch.from_numpy(X_train.astype(np.float32))
X_test_tensor  = torch.from_numpy(X_test.astype(np.float32))
y_train_tensor = torch.from_numpy(y_train.astype(np.float32))
y_test_tensor  = torch.from_numpy(y_test.astype(np.float32))

# ---- Model ----
class MySimpleNN(nn.Module):
    def __init__(self, num_features):
        super().__init__()
        self.linear = nn.Linear(num_features, 1)
        self.sigmoid = nn.Sigmoid()

    def forward(self, features):
        out = self.linear(features)
        out = self.sigmoid(out)
        return out

# ---- Train ----
model = MySimpleNN(X_train_tensor.shape[1])
loss_function = nn.BCELoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)

for epoch in range(25):
    y_pred = model(X_train_tensor)
    loss = loss_function(y_pred, y_train_tensor.reshape(-1, 1))
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    print(f"Epoch: {epoch+1}, Loss: {loss.item()}")

# ---- Evaluate ----
with torch.no_grad():
    y_pred = model(X_test_tensor)
    y_pred = (y_pred > 0.5).float()
    accuracy = (y_pred.squeeze() == y_test_tensor).float().mean()
    print(f"Accuracy: {accuracy.item()}")
→
This is the production pattern. Every PyTorch tutorial, course, and research paper follows this exact structure: define model class → set loss + optimizer → training loop (forward → loss → zero_grad → backward → step) → evaluate. Memorize this pattern.
